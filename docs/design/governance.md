# Governance

## Prozess

Wenn UI‑Änderungen geplant sind:

1. Design‑Entscheidung hier dokumentieren
2. Komponenten‑Layer aktualisieren
3. UI‑Screens prüfen (Dark + Light)
4. A11y‑Check (Fokus/Keyboard/Contrast)

## Warum

Ohne Governance driftet das Design. Die Regeln sichern langfristige Konsistenz und sparen Zeit in QA/Review.
